# Wakehood — Bankr agent skill

Wakehood is the decentralized index of AI trading agents on **Robinhood Chain (4663)**. Agents
trade native crypto **and** tokenized real-world assets (equities, ETFs). With this skill,
`@bankrbot` can search the index, read any agent or developer, and publish a new agent on the
user's behalf.

- **Site**: https://www.wakehood.com
- **Docs**: https://www.wakehood.com/docs
- **Explore**: https://www.wakehood.com/explore

Wakehood is a hosting and discovery platform, **not** a financial adviser and **not** an
execution engine. It stores and indexes agent code (on IPFS); it never runs it and never
custodies funds. Nothing on it is investment advice.

## What it is (and is not)

| Concept | Meaning |
|---|---|
| **Identity** | An Ed25519 keypair generated by the user. The public key IS the account. There is no email, no password on a server, no recovery. |
| **Agent** | A published trading strategy: name, description, asset class, strategy tags, and a pointer (IPFS CID) to its code. |
| **Asset class** | `CRYPTO`, `STOCK`, `ETF`, or `MIXED`. STOCK/ETF are tokenized real-world assets on Robinhood Chain. |
| **Trust score** | `+10` per published agent `+1` per day since the account was created. It measures participation and longevity, **NOT** trading skill, returns, or safety. Never present it as a quality signal. |
| **Wallet** | Optional and separate from identity. Not required to browse or publish. |

## Reading the index

### Search and filter agents

```bash
# free-text search across name + description
curl -s "https://www.wakehood.com/api/agents?q=arbitrage"

# filter by asset class (CRYPTO | STOCK | ETF | MIXED)
curl -s "https://www.wakehood.com/api/agents?assetType=MIXED"

# filter by strategy tag (url-encode spaces)
curl -s "https://www.wakehood.com/api/agents?tag=yield%20farming"

# combine
curl -s "https://www.wakehood.com/api/agents?assetType=CRYPTO&tag=momentum&q=breakout"
```

Response: `{ "agents": Agent[] }`, newest first. Each `Agent`:

```json
{
  "id": "clx...",
  "name": "Delta-Neutral Maxi",
  "description": "Harvests perpetual funding rates while staying delta-neutral...",
  "assetType": "CRYPTO",
  "tags": ["arbitrage", "market making"],
  "ipfsCid": "bafkrei...",
  "ownerPublicKey": "3f1a9c7e...7d2b48e1",
  "createdAt": "2026-07-14T...Z",
  "trustScoreContribution": 10
}
```

Link a user to an agent at `https://www.wakehood.com/agent/{id}` and to its code at the
gateway `https://<gateway>/ipfs/{ipfsCid}`.

### One agent, with its owner

```bash
curl -s "https://www.wakehood.com/api/agents/{id}"
```

Returns `{ agent, owner }` (404 if not found). `owner` includes the developer's `trust`
breakdown.

### A developer's profile

```bash
curl -s "https://www.wakehood.com/api/profile/{publicKey}"
```

Returns `{ profile }` with the developer's agents and `trust` (`{ total, fromAgents, fromDays,
agentCount, days }`). Any valid 64-hex public key resolves, even one that has not published yet
(empty profile), so this is safe to call for any key.

### Platform stats

```bash
curl -s "https://www.wakehood.com/api/stats"
# { "agentCount": 12, "developerCount": 2, "assetClassCount": 4 }
```

## Strategy tag taxonomy

The server rejects a publish whose tags fall outside the chosen asset class. Valid tags:

| Asset class | Allowed tags |
|---|---|
| `CRYPTO` | `arbitrage`, `momentum`, `market making`, `yield farming` |
| `STOCK` | `equity trading`, `ETF rebalancing`, `dividend strategy`, `sector rotation` |
| `ETF` | `ETF rebalancing`, `sector rotation`, `dividend strategy` |
| `MIXED` | the union of the CRYPTO and STOCK tags |

## Publishing an agent (signed)

Publishing is authenticated with the user's Ed25519 private key — there are no accounts or
tokens. The full, exact signing scheme is in [`references/SIGNING.md`](references/SIGNING.md).
Read it before implementing; getting the canonical message wrong yields a `401`.

**Flow:**

1. **Identity.** Reuse the user's Wakehood keypair, or generate one (Ed25519, 32-byte seed).
   The public key is 64-hex. Warn the user: the private key cannot be recovered — back it up.
2. **Code to IPFS.** `POST /api/ipfs` as `multipart/form-data` with `files[]` and a `name`,
   or supply an existing CID. Response: `{ cid, mock }`. (`mock: true` means IPFS is not
   configured on that deployment and the CID is a local placeholder.)
3. **Build the body** exactly:
   ```json
   { "name": "...", "description": "...", "assetType": "CRYPTO",
     "tags": ["momentum"], "ipfsCid": "bafkrei..." }
   ```
   `name` ≤ 80 chars, `description` ≤ 2000, at least one valid tag for the asset class.
4. **Sign** the canonical message for `method=POST`, `path=/api/agents`, `timestamp=Date.now()`,
   and this body (see references/SIGNING.md). Signature is hex.
5. **POST**:
   ```bash
   curl -s -X POST https://www.wakehood.com/api/agents \
     -H 'content-type: application/json' \
     -d '{ "publicKey":"<hex>", "signature":"<hex>", "timestamp":<ms>, "agent":<body> }'
   ```

**Responses:** `201 { agent }` on success · `401` bad/expired signature (the request must be
within 2 minutes of the server clock; the signer owns the agent) · `400` invalid fields ·
`409` the signer already has an agent with that name.

On success, show the user `https://www.wakehood.com/agent/{agent.id}`.

## Safety and honesty rules

- Wakehood **never executes** agent code. Do not imply an agent will run or trade for the user.
- Wakehood is **not** a financial adviser. Do not give investment advice or predict returns.
- The **trust score is not performance.** Never rank or recommend agents as "good" by trust.
- Third parties publish agents; they are **not** audited or endorsed. Tell users to review the
  code (via the IPFS link) themselves before using any agent.
- A lost private key is **unrecoverable.** Always tell the user to back it up before publishing.
- Wakehood is **not affiliated with Robinhood Markets, Inc.**
